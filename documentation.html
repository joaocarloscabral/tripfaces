Configuration

in tripfaces, we provide three three kind of options for developers to customize their own trips.

  1. tour tag properties - change the default behavior of tripfaces and would influence all trips.
  2. message options - only change the behavior of specifc trip and won't influence the others.

tour options

widgetVar

you can set widgetVar if you want start your trip manually.

  • type : String
  • default : trip

theme

you can set which theme to use, now tripfaces support white, black, yeti and dark 4 themes.

  • type : String
  • default :dark

autoStart

you can decide if your trip will start by itself (document ready).

  • type : Boolean
  • default : true

backToTopWhenEnded

you can ask tripfaces go back to top when ended.

  • type : Boolean
  • default : false

enableKeyBinding

you can decide to bind key events for trip navigations or not.

  • type : Boolean
  • default : true

enableAnimation

you can enable animations on tripBlock.

  • type : Boolean
  • default : true

showCloseBox

you can decide to add small close icon on each trip or not

  • type : Boolean
  • default : false

showHeader

ff you want to show no matter with default or customized header, you have to set this value to true. You can check header option for more details.

  • type : Boolean
  • default : false

showNavigation

Toggle navigation buttons

  • type : Boolean
  • default : false

header

this is the header for each trip. You can customize your own if you don't want to use default one. You can put {{tripIndex}} and {{tripTotal}} in the title to notify users about the number of trips and current index of trip. Later when rendering, tripfaces will replace them with right information.

  • type : String
  • default : step {{tripIndex}}

prevLabel

label for the previous button.

  • type : String
  • default : Prev

prevLabel

label for the next button.

  • type : String
  • default : Next

finishLabel

label for the finish button.

  • type : String
  • default : Done

animation

animation that would be applied on the tripBlock.

  • type : String
  • default : tada

onStart()

you can set a callback function triggered when tripfaces starts.

  • type : JavaScript code
  • default :

onEnd()

you can set a callback function triggered when tripfaces ends.

  • Type : JavaScript code
  • Default :

message tag properties

target

which id is referenced in this step

  • type: String
  • required: true

value [required]

what information that you want to show to users. You can put {{tripIndex}} and {{tripTotal}} in the content to notify users about the number of trips and current index of trip. later when rendering, tripfaces will replace them with right information.

  • type : String

position

what position would you prefer for the tripBlock. we support 4 different positions : e , w , n , s .

  • type : String
  • default : 'n'

expose

you can enable this option to expose your selected element in this trip.

  • type : Boolean or String (for id referring)
  • default : false

delay

every trip will be presented for 1 second (1000 ms) by default. set this value to -1 will prevent auto-advancing and you must manually call widgetVar.next() .

  • type : Number
  • default : 1000

canGoPrev

determines if the user can go backwards. if set to a function it is called and the response is evaluated.

  • type : Boolean
  • default : true

canGoNext

determines if the user can go forward. if set to a function it is called and the response is evaluated.

  • type : Boolean
  • default : true

animation

animation that would be applied on the tripBlock.

  • type : String
  • default : tada
